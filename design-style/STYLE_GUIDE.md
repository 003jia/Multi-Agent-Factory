# 多 Agent 工厂桌面客户端样式

## 定位

本样式用于“多 Agent 工厂”本地桌面客户端工作台。界面应像 Agent 编排控制台，而不是传统后台管理系统或营销页。

核心信息优先级：

1. 当前项目
2. 当前任务
3. 当前步骤
4. 责任 Agent
5. 模型与 API Key 是否可用
6. 验证门禁与下一步操作

## 视觉原则

- 简约、克制、工程化、专业。
- 主色为白色 + 蓝色，背景使用浅灰白。
- 不使用大面积渐变、暗黑背景、花哨插画。
- 卡片只用于承载明确的信息单元，不做层层嵌套。
- 圆角统一控制在 8px。
- 阴影只用于弹窗和重要浮层。

## 颜色

| Token | Value | Usage |
| --- | --- | --- |
| `background` | `#F8FAFC` | 主工作区背景 |
| `surface` | `#FFFFFF` | 侧栏、顶部栏、卡片、面板 |
| `surface-muted` | `#F5F7FB` | 次级浅灰背景 |
| `primary` | `#2563EB` | 主按钮、当前步骤、品牌标识 |
| `primary-soft` | `#EFF6FF` | 当前导航、选中步骤背景 |
| `primary-soft-border` | `#DBEAFE` | 蓝色弱边框 |
| `text-primary` | `#0F172A` | 标题、主要数据 |
| `text-secondary` | `#64748B` | 说明文字、元信息 |
| `text-muted` | `#94A3B8` | 占位、低优先级信息 |

| `success` | `#16A34A` | 通过、已完成、正常 |

## 布局

- 画布基准：`1440 x 900`
- 左侧导航栏：`260px`
- 顶部任务栏：`68px`
- 主区域：当前任务工作区
- 右侧 Inspector：约 `342px`
- 主内容与 Inspector 间距：`24px`
- 页面内基础间距：`8 / 12 / 14 / 24`

## 组件约定

### 左侧导航

包含：

- 工作台
- 项目
- 任务
- 智能体
- 子智能体
- 审查队列
- 设置

当前选中项使用 `primary-soft` 背景与 `primary` 文本。

### 顶部任务栏

只展示摘要状态：

- 当前项目
- 当前任务
- 模型状态
- API Key 状态
- 搜索入口

不直接铺开 Base URL、模型名等高级配置。

### 任务流程管线

7 步作为任务详情中的流程状态，而不是全局页面导航：

1. 任务输入
2. 需求文档
3. 任务计划
4. 智能体分配
5. 代码生成
6. 代码审查
7. 应用完成

当前步骤用蓝色强调；已完成为绿色点；等待或处理中为黄色点；阻断为红色点。

### Work Item 卡片

任务计划页优先使用 Work Item 卡片，不用大段 textarea。

每张卡片包含：

- 编号
- 标题
- 描述
- 目标文件
- 责任 Agent
- 模型档位
- 验收标准
- 验证命令
- 当前状态

### 右侧 Inspector

根据当前步骤动态变化，常见模块：

- 项目上下文
- 责任 Agent
- 风险门禁
- 模型策略
- 反馈消息

### API Key 设置弹窗

默认只展示：

- Provider 选择
- API Key 输入框
- 保存按钮
- 测试连接按钮
- 当前状态

高级设置折叠展示：

- Base URL
- economy 模型
- quality 模型
- 清除 Key
