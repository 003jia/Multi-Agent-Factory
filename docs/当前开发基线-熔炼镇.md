## 产品方向

当前产品不再以“大工作台默认展开全部功能”为主，而以“熔炼镇”作为统一隐喻：

- 用户任务是原矿。
- 模型和本地 Agent 运行时是炉火。
- 需求、计划、代码、审查、提交是镇里的工坊路线。
- 首页保持一个主要任务输入入口。
- 高级能力进入任务详情或设置，不抢首页主体验。

## 首页必须保留的结构

1. 顶部品牌：`熔炼镇 / 多 Agent 工厂`。
2. 炉火状态：真实 AI、mock、Codex/Claude Code 运行时状态需要明确区分。
3. 今日炼制：用少量指标展示完成、进行、等待、错误。
4. 炼制流程：需求炉、计划坊、铸码间、审查塔、提交仓。
5. 任务输入框：中心主入口。
6. 原矿配方：任务模板。
7. 熔炼队列或近期运送：最近任务与阶段。

## 当前工程映射

- React 入口：`src/App.tsx`
- 样式：`src/styles.css`
- AI API 引擎：`server/agents/openaiEngine.ts`、`server/agents/claudeEngine.ts`
- Mock 引擎：`server/agents/mockEngine.ts`
- 运行时状态接口：`/api/health`
