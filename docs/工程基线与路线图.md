## 一、当前真实状态（已实测）

`typecheck` exit 0 · `npm test` **28 通过 / 8 文件** · `build` ok · server+src 约 **6098 行**。

它现在实际是：一个 **Electron 桌面工作台**（Web 同源可跑），完成 7 步炼制流程——

> 扫描真实项目 → 选文件/给约束 → 生成需求 → 确认 → 生成计划 → 确认 → 拆 work item + 语义分派 → 生成补丁集(PatchSet) → 沙箱跑真实 `tsc` 校验 → 审查门禁 → 应用补丁到**真实磁盘** → 提交/应用记录；问题可回流到责任 Agent 修复重跑。

AI 支持 **Anthropic 与 OpenAI 兼容格式**（可配 baseUrl，端到端含设置页 UI + 本地持久化）；无 Key 时降级 **Mock**（离线、测试确定）。

## 三、Agent 角色与分工（吸收 Gas Town 思想）

Gas Town 的价值不在复制 CLI/术语，而在把多 Agent 编排当成**可追踪的运行系统**：明确角色、持久化工作单、容量调度、审查门禁、问题升级。当前 8 个 subagent（[mockAgents.ts](../server/mockAgents.ts)）：

| Agent | 职责 | 默认档 |
|-------|------|--------|
| 协调调度 | 接目标、推进 7 步、维护交接与分派 | quality |
| 产品需求 | 生成/更新需求文档 | economy |
| 任务计划 | 拆 work item、定义门禁与依赖 | quality |
| 前端实现 / 后端实现 | 生成可审查代码产物 | economy |
| 代码审查 | 质量门禁，输出审查问题 | quality |
| 问题修复 | 接回流，定位责任 work item 并修复 | quality |
| 监督巡检 | 监控容量/失败/卡住与升级 | economy |

**分派**：[scoreSubagent](../server/mockAgents.ts) 按"技能 ∩ work item（标题/描述/`targetFiles`）+ 文件领域 + 风险偏好 premium + 元角色不接实现单 + 并发负载"打分（已替换旧的正则匹配）。

## 五、路线图（按优先级）

### 核心命脉——把"多 Agent"做成真的（北极星，尚未动）
- **M2 · 依赖产物交接**：`WorkItem.dependencies` 已填但**生成时未消费**。生成某 work item 时注入其依赖项已生成产物的摘要，实现 agent 间"读彼此产出"。
- **M3 · Agent 真用工具迭代**：当前每个 agent 仍是**单次 LLM 调用披前缀**（`grep tool_use` 零命中）。改成 tool-use 循环——agent 能 read 项目文件/依赖产物、write/edit、沙箱跑 verify，按编译错误自我迭代到通过。Anthropic 用 SDK tool runner，OpenAI 用 function calling，对外仍是统一 `AgentEngine`。
- **M4 · 协调者(Mayor)**：用 LLM 把 plan 拆成带依赖图的 work items + 初分派，替代模板拆分。

### 一致性收尾（小、立刻消歧义）
- **apply/submit 语义统一**：[app.ts](../server/app.ts) 两路由都调 `workflow.submit`，且 `submit` 实际**写真实磁盘**。建议：落盘动作命名 `apply` + 加显式确认；"提交记录"与"落盘"分清。
- **演示模式也做静态校验**：默认"不绑定项目"时 [workflow.ts](../server/workflow.ts) + projectPatch **跳过沙箱校验** → P0"坏代码被抓"的保证默认关闭。建议无项目时也对产物做隔离静态校验（P0 原本就能）。
- **建立 git 基线**：当前 6098 行代码**零提交**，无回滚点。建议把当前绿状态提交为基线。

## 六、不变量（不能被绕过）

- 后端状态机阻止越级：未确认计划不能进入分派。
- 审查存在开放 `error` 时**不能提交/落盘**。
- 回流必须记录责任 finding / work item / subagent / note。
- 写真实磁盘前必须通过沙箱校验且无被阻止的变更集（hash 校验保护）。
- 无 Key 时降级 Mock，且 UI 必须明确区分"真实 AI / Mock / 运行时"，不假装在干活。
